## 1. 项目背景与核心价值

### 1.1 现实痛点

中国已进入深度老龄化社会。老年人在阅读药品说明书时面临几个核心困难：

1. **字体细小**: 药品说明书为了节省成本，字体通常很小
2. **阅读困难**: 老花眼、白内障等视力问题导致阅读障碍
3. **专业术语**: "适应症"、"禁忌症"、"不良反应"等医学术语晦涩难懂

这直接导致用药安全风险：误服、漏服、剂量错误等问题频发。

### 1.2 解决方案价值

本系统通过"拍照→识别→播报"的极简交互，让老年人：

- 无需佩戴老花镜即可获取药品信息
- 无需理解专业术语，系统自动转化为通俗语言
- 通过语音播报，解放双眼，降低认知负担

从而降低老年人用药安全风险，提升医疗可及性，并展示模型在 OpenVINO 上的完整部署方案

## 2. 现有系统解决方案概述

### 2.1 系统整体架构

当前系统采用三阶段管线架构，实现从药品说明书图片到语音播报的完整流程：

```
药品说明书图片输入
      ↓
[阶段1] OCR文字识别
      ↓
原始OCR文本
      ↓
[阶段2] 多模态信息提取
      ↓
结构化关键信息
      ↓
[阶段3] 语音合成
      ↓
语音音频输出
```

### 2.2 核心模型与技术栈

#### 2.2.1 OCR文字识别阶段

**使用模型**: PaddleOCR-VL-1.5（`megemini/PaddleOCR-VL-1.5-OpenVINO`）

**处理流程**:

1. 图片输入与预处理
2. 可选的图片分割策略（应对小字体场景）
3. OCR模型推理识别

#### 2.2.2 多模态信息提取阶段

**使用模型**: Qwen3-VL-4B-Instruct（`snake7gun/Qwen3-VL-4B-Instruct-int4-ov`）

**处理流程**:

1. 将OCR识别文本和原图作为输入
2. 通过精心设计的提示词引导模型提取关键信息
3. 提取5项核心信息：药品名称、适应症、用法用量、禁忌、不良反应
4. 将专业术语转化为通俗易懂的语言

#### 2.2.3 语音合成阶段

**使用模型**: Qwen3-TTS-CustomVoice-0.6B（`snake7gun/Qwen3-TTS-CustomVoice-0.6B-fp16-ov`）

**处理流程**:

1. 将提取的关键信息文本作为输入
2. 选择说话人风格（如"vivian"）
3. 合成自然流畅的语音音频
4. 输出音频文件

### 2.3 关键技术设计

#### 2.3.1 懒加载内存优化

**问题**: 三个模型同时加载需要大量内存

**解决方案**:

- 采用懒加载策略，模型按需加载
- 每个阶段完成后释放前一阶段模型
- 显著降低峰值内存占用

#### 2.3.2 图片分割策略

**问题**: 药品说明书字体太小，直接识别准确率低

**解决方案**:

- 将图片分割为多个子图（如2x2、3x3网格）
- 相邻子图保留重叠区域，避免文字截断
- 分别识别后合并结果

## 3. 现有系统的问题分析

### 3.1 单张图片处理的局限性

实际药品说明书存在多种复杂情况

- 大多数药品说明书是双面印刷，少数采用多页的方式
- 正面通常包含：药品名称、成分、适应症
- 反面通常包含：用法用量、不良反应、禁忌、注意事项

**现有问题**:

- 用户只能拍摄上传一张图片，导致信息不完整
- 如果只拍正面，缺少关键的用法用量和禁忌信息
- 如果只拍反面，不知道是什么药

**影响**: 信息提取不完整，可能导致用药指导错误

### 3.2 图片质量问题的处理不足

很多药品说明书尺寸很大，但是字体很小

**现有问题**:

- 用户使用 *高像素* 手机拍摄
- 直接输入大尺寸图片，OCR模型处理困难
- 一方面尺寸大，另一方面字体小，更加识别困难
- 大尺寸图片导致内存溢出
- OCR识别速度极慢

**影响**: 系统响应慢，用户体验差，甚至无法完成识别

### 3.3 多模态模型能力不足

pipeline 第二步使用 VLM 多模态模型整理文字信息

**现有问题**:

- 理论上可以省去第一步的 OCR 识别，但是，实际情况是，面对药品说明书这类大尺寸小字体的情况，VLM 模型的识别率很差
- VLM 输入 OCR 和图片信息后进行推理需要很大的内存消耗

**影响**: 系统响应慢，识别率低

### 3.4 语音合成的文本不顺畅

pipeline 第三步将 VLM 整理的文本转换为语音

**现有问题**:

- 输入的文本断句不完整
- 存在部分特殊符号影响 TTS

**影响**: 语音合成不自然，不顺畅

## 4. 改进方案设计

### 4.1 多图片输入支持

可以上传多张图片，系统逐张图片提取信息，通过 VLM 进行信息整合与抽取

### 4.2 图片质量优化

- 检测图片尺寸，自动缩放并保持宽高比，避免变形。
- 参考 PaddleOCR 中预处理方法对图片进行图像预处理增强

### 4.3 优化 pipeline 衔接

- 限制 VLM 输出的长度
- 优化 VLM 输出符合 TTS 合成的风格
